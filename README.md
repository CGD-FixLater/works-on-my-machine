# works-on-my-machine

我的配置与折腾记录合集 —— 每搞定一套环境/工具的配置，就整理成「踩坑笔记 + 一键脚本」放进来，方便换机器复用，也分享给可能踩同样坑的人。「在我机器上能跑」，希望在你机器上也能跑。

每个主题一个子文件夹，通常包含：
- 一份 Markdown 记录：讲原理、分步操作、踩过的坑、回滚方法。
- 一个一键脚本：想偷懒可直接跑。

## 目录

| 配置 | 适用环境 | 作用 |

> 以后每加一套配置，在上表追加一行即可。

## 使用

进对应文件夹，按里面的 `README` 或 `*.md` 操作；多数主题也提供一键脚本，例如：

```bash
cd rime-ice-setup
bash install-rime-ice.sh
```

## 新增一套配置

每次折腾完一套环境，按这个流程归档：

1. **建子文件夹**，名字用 `主题-用途`，如 `rime-ice-setup`、`zsh-setup`。
2. 放进**两样东西**（脚本可选）：
   - 一份 Markdown 记录：环境、分步操作、**踩过的坑**、回滚方法。
   - 一个一键脚本（能自动化就配，不强求）。
3. **在上面「目录」表格加一行**，写清适用环境和一句话作用。
4. 提交并推送：
   ```bash
   git add .
   git commit -m "add: <主题> 配置记录"
   git push
   ```

写记录时的几条习惯（让别人——和未来的自己——少踩坑）：

- **写清适用环境**（系统版本、桌面、关键软件版本），方案常常和版本强相关。
- **把踩过的坑单独标出来**，这往往比正常步骤更值钱。
- **给回滚方法**，让人敢动手。
- **提交前脱敏**：别带 token、密钥、邮箱、内网地址。`.gitignore` 已挡掉常见敏感文件，但仍要自己过一眼。
